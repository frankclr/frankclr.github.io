Windows

组策略配置

  1. 运行gpedit.msc
    1. 计算机配置——》管理模板——》
      1. Windows组件——》Windows更新——》管理从Windows 更新提供的更新 【更新管理】
        1. ——》Windows更新不包括驱动程序——》已启用 【禁用驱动更新】
        2. ——》选择何时接收质量更新——》已启用 ——》延期天数：30 【避免更新Bug】
        3. ——》选择何时接收预览版和功能更新——》已启用 ——》延期天数：90 【避免更新Bug】
      2. 系统 –> 文件系统 –> 启用 Win32 长路径 【长路径】

以上设置好后，再连接网络

开发人员设置

Windows 11 24H2版本后共享设置

Windows 11 24H2版本更改了 SMB 共享默认设置。求强制开启 SMB 签名（以提高安全性）

专业版

设置组策略

Win + R –> gpedit.msc –> 计算机配置

  1. –> Windows设置 –> 安全设置 –> 本地策略 –> 安全选项
    1. –> 禁用“Microsoft网络客户端：对通讯进行数字签名（始终）”【客户端:访问其他电脑的共享】
    2. –> 禁用“Microsoft网络服务器：对通讯进行数字签名(始终)”【服务器:本机还要作为共享服务的服务器，比如要将文件夹共享给区域网内的其他设备使用】
  2. –> 管理模板 –> 网络 –> Lanman工作站 –> 启用"启用不安全的来宾用户"【服务器:开启匿名登录,否则匿名登录不可用，即使服务器允许也不可用】

重启电脑，并重设文件夹共享(不重设就一直报错)
